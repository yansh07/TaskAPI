The System Design

Not just a CRUD app. A complete distributed event loop handling jobs, retries, and dead-letter queues.

Atomic Locking

Prevents race conditions using Redis scripts. We guarantee a task is picked up by exactly one worker, even under heavy load.

SET task_id LOCKED NX EX 30

DAG Workflows

Complex dependency resolution. Job B only runs if Job A succeeds. Supports topological scheduling for multi-stage pipelines.

Task(A) >> [Task(B), Task(C)]

Full Observability

We don't guess; we measure. OpenTelemetry traces for every request, Prometheus metrics for throughput, and Grafana for the pretty charts.

latency_bucket{job="resize"}

Resilience Patterns

Exponential backoff retries for transient failures. Poison pill handling via Dead Letter Queues (DLQ). Nothing gets lost.

Idempotency

Exactly-once(ish) execution guarantees. We use unique idempotency keys to deduplicate jobs within a configurable window.

Scalable Infrastructure

Horizontally scalable workers. Nginx reverse proxy. Dockerized for deployment. Ready for K8s if you're feeling brave.
